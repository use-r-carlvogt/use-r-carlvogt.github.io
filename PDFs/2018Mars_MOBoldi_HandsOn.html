A random effect linear model

During an afterwork party, you have been clever enough to be involved in the study of the rotational speed of eggs. This fascinating experience is performed by four top ranked scientists (A, B, C, D), including you, on a wooden table. An initial speed of 3 rps (3 round per second) is given to the egg. The speed is measured after 10 seconds. The results are given below1.

## 16  D  1.30
## 17  D  2.60
## 18  D  2.00
## 19  D  2.20
## 20  D  1.60

The model this experience is \[ y_{ij} = \mu + s_i + e_{ij}, \quad i=1,\ldots,4, \; j=1,\ldots,5. \] There, \(\mu\) is the average rotational speed of an egg, and \(s_i\) is the effect of subject \(i\) (scientist). There are 5 replicates per subject.

Since the subjects are not of interest we dont want to compare \(A\) to \(B\) for example but since we need to take their effect into account, they are fitted as random effects in the model. For linear models, normal random effects and residuals are assumed \[ s_i \stackrel{i.i.d.}{\sim} N(0, \sigma_s^2), \quad e_{ij} \stackrel{i.i.d.}{\sim} N(0, \sigma^2). \] The model is fitted to the data using package lme4 and lmerTest2.

require(lme4)
require(lmerTest)
mod <- lmer(Speed ~ 1 + (1|ID), data=dat1)
summary(mod)

The table is a random effect because table are taken at random, and not of interest3. The two random effects are crossed and independent. Try this:

mod4 <- lmer(Speed ~ Type + (1|Table) + (1|ID), data=dat4)
summary(mod4)

It reads as one intercept + Type effect + one intercept per Table + one intercept per ID.

The ANOVA F-test is for the fixed effect. Testing the random effects can be done with a likelihood ratio test using function rand on the model. Try and conclude.

rand(mod4)

Note: F-test and specially the Likelihood Ratio Test for mixed linear models must be used with care. The F-tests can be exact (even for random effect testing). The LRT is always approximate and can be severly biased.

## 16  C  E2  1.70

Since Egg E1 for researcher A is not the same as Egg E1 for researcher B (and so on), the Egg effect (which is random) is called nested (within effect ID). This design is noted as below. Produce the summary and the test of the random effect.

mod5 <- lmer(Speed ~ 1 + (1 | ID/Egg), data=dat5)

It reads as one intercept + one intercept per ID + one intercept per Egg within ID.

Note that, there is nothing in the data which indicates whether the effect is nested or not. If the egg would have been the same among researchers, then the two effects would have been crossed like for Table.

A nested effect can be seen as an interaction between two factors: the Egg effect (E1, E2) is changing from one ID to another (A, B, C, D).

Calling (1 | ID/Egg) in lmer includes both the effect ID and the interaction Egg:ID. This can be seen on summary(mod5). However, calling rand(mod5) only produce one global test. Suppose you would like to separate the main effect ID from the interaction Egg:ID, just call

mod5 <- lmer(Speed ~ 1 + (1 | ID) + (1 | Egg:ID), data=dat5)

You can check that the summary is the same. Function rand will separate the two tests. You can also use this if you want to include only the Egg effect without the ID effect (like it could be the case here).

Check out the result.

Back to the analysis, we see that the slope (coefficient associated to age in a regression) varies

  1. Per Sex: interaction age:Sex; the classical interaction between fixed effects in a linear regression.
  2. Per Subject: interaction age:Subject. This is an interaction between a fixed effect and a random effect. With a slope, it is called a random slope.

First lets ignore the Sex effect to simplify and focus on the Male data. To fit the model, use

mod6 <- lmer(distance ~ age + (age|Subject), data=Orthodont, subset=(Sex=="Male"))

Checkout for the summary of the model.

The formula is read as age is the fixed effect (like in a linear regression), and (age|Subject) means there is one age coefficient per subject. The corresponding mathematical model is \[ y_{ij} = \alpha+a_i + (\beta+b_i) t_{j} + \epsilon_{ij} \] where

  1. \(\alpha\) is the intercept and \(\beta\) is the average slope. \(a_i\) is the intercept gap of subject \(i\) (i.e. \(\alpha+a_i\) is the intercept of subject \(i\)).
  2. \(b_i\) is the slope gap of subject \(i\) (i.e. \(\beta + b_i\) is the slope of subject \(i\)).
  3. \(t_j\) is the age measured at occasion \(j\)
  4. \(\epsilon_{ij}\) is the residual.

The distributional hypotheses are \[ a_i \stackrel{i.i.d.}{\sim} N(0,\sigma_a^2), \quad b_i \stackrel{i.i.d.}{\sim} N(0,\sigma_b^2), \quad cor(a_i, b_i) = \rho, \quad \epsilon_{ij} \stackrel{i.i.d.}{\sim} N(0,\sigma^2) \] Try to identify on the summary(mod6) these coefficients.

In particular, youll see that \(\hat{\rho}=-0.79\). How can this be interpreted? In fact, by defaut, lmer assumes a correlation between random intercept and random slope. A strong negative correlation like this means that when \(a_i\) is large, then \(b_i\) is small.

Lets admit that often it is of low interest, and probably with risks of overfitting. It is possible to force \(\rho=0\) using

mod7 <- lmer(distance ~ age + (age || Subject), data=Orthodont, subset=(Sex=="Male"))

Check out the result.

It should be said that, like for the regression, we did not ask for a random intercept in the formula, i.e.we did not need to write 1+age. The 1 is implicit. Like for fixed effect, if we want to force the random intercept out, we need to expicitla write

mod8 <- lmer(distance ~ age + (0 + age | Subject), data=Orthodont, subset=(Sex=="Male"))

This explains why another way to have independent random intercept and slopes is to write (equiv. to mod7 above)

mod7 <- lmer(distance ~ age + (1|Subject) + (0+age | Subject), data=Orthodont, subset=(Sex=="Male"))

In both model, the observational independent units are \(y_1,\ldots,y_{10}\) where \(y_i = (y_{i1},y_{i2},y_{i3})\). Under the MANOVA model (multivariate linear model), the variance matrix is general: \[ var(y_i)=\left[ \begin{array}{c} \sigma_1^2 & \sigma^2_{12} & \sigma_{13}^2\\ * & \sigma_2^2 & \sigma^2_{23} \\ * & * & \sigma_3^2 \end{array} \right] \] Under the repeated measure model, the variance matrix is compound symmetric: \[ var(y_i)=\left[ \begin{array}{c} \sigma^2+\sigma_s^2 & \sigma^2_s & \sigma_s^2\\ * & \sigma^2+\sigma_s^2 & \sigma^2_s \\ * & * & \sigma^2+\sigma_s^2 \end{array} \right] \] In particular, all the observations \(y_{ij}\) have the same variance, and within-subject observations have the same correlation. This hypothesis is linked to sphericity (in fact it is stronger than). It can be shown that the F-test is valid under sphericity. However, it can be shown that a departure from this hypothesis can have strong impact on the real significant level of the test. On the other hand, using a MANOVA when sphericity is true provides a test with a lower power than ANOVA with repeated measures.

Mauchlys test can be used to test sphericity. Also the Greenhouse-Geisser correction, and the Huynh-Feldt corrections can be used to the F-test under non-sphericity.

The circumference \(g\) is modeled as a function of the tree age \(x\) by \[ g(x; \phi_a, \phi_h, \phi_s) = \frac{\phi_a}{1+\exp\{-(x-\phi_h)/\phi_s\}}. \] The parameters (to be estimated) are

  1. \(\phi_a\), the asymptotic circumference,
  2. \(\phi_h\), the age at which the tree attains half of its asymptotic circumference,
  3. \(\phi_s\), the growth scale.

This function is already implemented in lme4 and named SSlogis.

Without taking the tree effect into account, one could fit a non-linear model

Mixed models: Prediction

Prediction of random effects

In the previous example, we have extracted the prediction of the random components of the model. For a linear model, the component predictions can be extracted using function ranef

ranef(mod)
## $ID
##   (Intercept)
## A -0.06709585
## B  0.02400268
## C -0.03896248
## D  0.08205566

The prediction is the conditional expectation of the random effect given the data \(E(U|Y=y)\) in a generic mixed linear model \[ Y = X\beta + ZU + \epsilon. \] In the case of linear mixed models, the conditional distribution of \(U|Y=y\) is Gaussian and the prediction can be obtained by matrix straightforward multplications. In the case of non-linear mixed models (Orange tree example), the distribution of \(Y\) is not Gaussian anymore. The prediction is not more approximate.

Exercise: compute (extract using R) the predictions of the random components of all the previous models. Have a more detailed look at the crossed effects and the hierarchical models, and try to understand the structure of the random components by comparing them.

Predictions of the response

Considering model mod. The predictions are the sum of the estimated fixed effects and the predicted random effects. Compare

coef(summary(mod))[1]+ ranef(mod)$ID
predict(mod)

This corresponds to the plug-in formula \[ \hat{y} = X\hat{\beta} + X\hat{u} \] where \(\hat{\beta}\) is the estimate of the fixed effects, and \(\hat{u}\) the prediction of the random effects.

For the prediction of a new subject (a new researcher), the prediction of the random effect is 0. Compare

coef(summary(mod))[1]
predict(mod, newdata=data.frame(ID="E"), allow.new.levels = TRUE)

Model diagnostics

Assumptions for the model validity can be checked using diagnostics on the residuals and the random effects. Exploration can be done using plot (method plot.merMod in package lme4) and qqmath (in package lattice). To plot the model Pearsons residuals versus the fitted values

plot(mod2, resid(., type="pearson")~fitted(.))

The above corresponds to the default (i.e.plot(mod2)). To split it per researchers,

plot(mod2, resid(., type="pearson")~fitted(.) | ID)

To check the normality of the residuals using a QQ-plot

library(lattice)
qqmath(mod2)

The id option allow to identify the residuals larger than the normal quantile at level 1-id/2. For example,

qqmath(mod2, id=0.3)
